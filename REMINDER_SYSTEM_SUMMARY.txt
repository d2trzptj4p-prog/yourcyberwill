✅ WHAT WAS BUILT
─────────────────────────────────────────────────────────────────────────────

An automated hourly reminder system that:
• Checks all users with active check-ins every hour
• Identifies users with less than 50% of their check-in time remaining (< 45 days out of 90)
• Sends personalized reminder emails via Resend
• Includes time remaining (e.g., "15 days and 8 hours left")
• Shows percentage of time used (e.g., "52% used")
• Provides one-click access to dashboard
• Ensures only one reminder per check-in cycle

🔧 HOW IT WORKS
─────────────────────────────────────────────────────────────────────────────

TRIGGER:    Hourly cron job (pg_cron in Supabase)
            Runs at: 0 * * * * (every hour at :00)

QUERY:      Finds users where:
            • check_in_active = true
            • reminder_notification_sent = false
            • check_in_due_at is in the future
            • Time remaining < 45 days (50% of 90-day window)

EMAIL:      Sends via Resend API with:
            • Subject: "⏰ Reminder: Check in soon – X% time remaining"
            • Personalized greeting (user's full name)
            • Time remaining formatted: "X days and Y hours"
            • Percentage of time used
            • Direct link to dashboard
            • Professional branding

UPDATE:     Marks user as reminded:
            • reminder_notification_sent = true
            • last_reminder_notification_at = current timestamp

RESULT:     Each user gets exactly ONE reminder per 90-day cycle


📊 EXAMPLE TIMELINE
─────────────────────────────────────────────────────────────────────────────

Day 0:      User starts 90-day check-in
            • check_in_due_at = Day 90
            • reminder_notification_sent = false

Day 45:     Threshold crossed (50% remaining)
            • No action yet, waiting for cron

Hour X:     Cron job executes (within 1 hour of day 45)
            • Sends reminder email
            • Email says: "You have 45 days left"
            • Sets reminder_notification_sent = true

Days 46-89: Reminder already sent
            • Function skips this user (flag is true)

Day 90:     Check-in overdue
            • daily-check-in function notifies recipients
            • Vault access granted to recipients

    └─ true when reminder has been sent for current cycle

🛠️ CUSTOMIZATION
─────────────────────────────────────────────────────────────────────────────

CHANGE REMINDER THRESHOLD:
  File: supabase/functions/check-in-reminder/index.ts
  Line: const FIFTY_PERCENT_THRESHOLD = CHECK_IN_INTERVAL_MS / 2;
  → Change divisor to customize threshold (currently 2 = 50%)

CHANGE CRON SCHEDULE:
  File: supabase/migrations/20260601000001_schedule_check_in_reminder.sql
  Line: '0 * * * *'
  → Change cron expression:
    '0 9 * * *'     = Daily at 9 AM UTC
    '0 */6 * * *'   = Every 6 hours
    '0 */4 * * *'   = Every 4 hours
    '0 8-20 * * *'  = Every hour 8 AM - 8 PM UTC

CUSTOMIZE EMAIL TEMPLATE:
  File: supabase/functions/check-in-reminder/index.ts
